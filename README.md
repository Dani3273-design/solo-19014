# 记忆翻牌游戏

一个使用 Pygame 开发的经典记忆翻牌游戏，支持中文显示，界面美观，操作流畅。

## 游戏说明

### 游戏规则
1. 这是一个经典的记忆翻牌游戏，测试你的记忆力
2. 游戏中有 4×4 = 16 张卡片，共 8 种不同的图案
3. 每种图案有两张相同的卡片，玩家需要找出所有配对的卡片
4. 每次只能翻开两张卡片：
   - 如果图案相同 → 配对成功，卡片保持翻开状态并置灰
   - 如果图案不同 → 短暂显示后自动翻回背面
5. 用最少的步数和时间完成游戏！

### 游戏界面
1. **初始页面**：显示游戏标题、游戏说明（左对齐）和"开始游戏"按钮
2. **游戏页面**：
   - 4×4 排列的卡片，初始背面朝上
   - 底部显示当前步数和耗时
   - 每次随机生成卡片位置
3. **胜利页面**：
   - 显示胜利信息
   - 显示总步数和总耗时
   - "重新开始"按钮，点击可重新游戏

### 图案说明

- 🍎 苹果

## 安装说明

### 环境要求
- Python 3.9 或更高版本
- pip 包管理器

### 安装依赖

1. 首先确保已安装 Python 3.9+
```bash
python3 --version
```

2. 安装 Pygame
```bash
pip3 install pygame
```

或者使用 requirements.txt（如果已创建）：
```bash
pip3 install -r requirements.txt
```

### 运行游戏

进入项目目录，运行 main.py：

```bash
cd main
python3 main.py
```

### 模块说明

#### 1. ui.py (UI 模块)
- `GameState`: 游戏状态枚举（菜单、游戏中、游戏结束）
- `Card`: 卡片类，负责卡片的绘制、点击检测
- `Button`: 按钮类，支持悬停效果和点击回调
- `UIManager`: UI 管理器，负责所有页面的渲染和用户输入处理

主要功能：
- 初始菜单页面渲染（标题、左对齐的游戏说明、开始按钮）
- 游戏页面渲染（4×4 卡片布局、步数和时间显示）
- 胜利页面渲染（统计信息、重新开始按钮）
- 卡片图案随机化
- 鼠标移动和点击事件处理

#### 2. control.py (控制模块)
- `GameController`: 游戏控制器，管理游戏核心逻辑

主要功能：
- 处理卡片点击事件
- 管理选中卡片的状态
- 判断卡片是否匹配：
  - 匹配成功：标记为已匹配，置灰显示
  - 匹配失败：使用多线程延迟翻回
- 多线程处理：
  - 配对失败时使用独立线程延迟 1 秒后翻回卡片
  - 不阻塞主 UI 线程，保证操作流畅
- 使用 `threading.Event` 进行线程同步

#### 3. engine.py (引擎模块)
- `AnimationThread`: 动画线程基类
- `FlipAnimation`: 翻转动画类
- `Particle`: 粒子类
- `ParticleSystem`: 粒子系统
- `GameEngine`: 游戏引擎主类

主要功能：
- 多线程动画框架，支持独立的动画线程
- 翻转动画效果（可选使用）
- 粒子特效系统：
  - 支持重力模拟
  - 支持生命周期管理
  - 支持透明度渐变
  - 配对成功时可产生金色粒子爆炸效果
- 所有特效处理使用线程安全机制

#### 4. main.py (入口模块)
- 初始化 Pygame
- 创建游戏窗口 (600×700 像素)
- 实例化各模块并协调工作
- 主游戏循环：
  - 事件处理（鼠标移动、点击、窗口关闭）
  - 状态更新
  - 画面渲染
  - 帧率控制 (60 FPS)

## 技术特性

### 1. 中文支持
- 使用 macOS 系统字体 `PingFang.ttc` 渲染中文文本
- 自动降级机制：如果系统字体不可用，使用 pygame 默认字体
- 测试过中文标题、说明文字、按钮文字的正常显示

### 2. UI 设计
- 采用暖色调配色方案（米色背景、金色按钮、蓝色卡片背面）
- 圆角矩形按钮和卡片，现代化视觉效果
- 按钮悬停效果（颜色变化）
- 卡片边框和阴影效果
- 信息栏背景带边框
- 所有元素布局居中对齐，无重叠错位

### 3. 多线程设计
游戏使用多线程确保操作流畅，不卡顿：

#### 线程使用场景：
1. **卡片翻回动画**：
   - 当两张卡片不匹配时，需要延迟 1 秒后翻回
   - 使用 `threading.Thread` 创建守护线程
   - 线程内部使用 `time.sleep(1.0)` 等待
   - 等待期间主 UI 线程继续运行，响应其他操作（如鼠标悬停）

2. **线程同步机制**：
   - 使用 `threading.Event` 作为锁信号
   - `is_locked` 标志防止重复点击
   - 线程安全的状态访问

#### 为什么使用多线程：
- 如果不使用多线程，`time.sleep()` 会阻塞整个主线程
- 阻塞期间：窗口无响应、鼠标悬停效果失效、画面冻结
- 使用多线程后：等待翻回的 1 秒内，UI 仍然流畅响应

### 4. 代码架构
- 模块化设计：UI、控制、引擎分离
- 面向对象编程：每个功能封装为类
- 状态机模式：通过 `GameState` 管理不同游戏阶段
- 观察者模式：按钮使用回调函数机制

### 5. 游戏特性
- 卡片位置每次游戏随机生成
- 匹配成功的卡片置灰显示，区分未匹配卡片
- 步数统计（每翻开两张卡片计为一步）
- 时间统计（精确到秒）
- 支持重新开始游戏

## 操作说明

### 鼠标操作
1. **左键点击卡片**：翻开卡片
2. **左键点击按钮**：触发按钮功能
3. **鼠标移动**：按钮悬停时显示高亮效果

### 游戏流程
1. 启动游戏 → 进入初始页面
2. 阅读游戏说明（左对齐显示）
3. 点击"开始游戏"按钮 → 进入游戏页面
4. 点击卡片翻开，寻找相同图案：
   - 第一张卡片翻开，等待选择第二张
   - 第二张卡片翻开，自动判断是否匹配
   - 匹配成功：两张卡片保持翻开并置灰
   - 匹配失败：1秒后自动翻回
5. 所有卡片匹配完成 → 进入胜利页面
6. 查看步数和耗时
7. 点击"重新开始"按钮 → 重新开始游戏

## 已知问题与解决方案

### 问题 1：Emoji 字体显示
在某些系统上，emoji 可能无法正常显示为彩色。

**解决方案**：
- 代码已优先尝试加载 `Apple Color Emoji.ttc` 字体
- 如果失败，降级使用 pygame 默认字体
- 建议在 macOS 上运行以获得最佳 emoji 显示效果

### 问题 2：中文方块
在非 macOS 系统上，中文可能显示为方块。

**解决方案**：
- 修改 `ui.py` 中的 `_init_fonts` 方法
- 将字体路径改为对应系统的中文字体路径
- Windows: `C:/Windows/Fonts/msyh.ttc` (微软雅黑)
- Linux: `/usr/share/fonts/truetype/droid/DroidSansFallbackFull.ttf`

### 问题 3：窗口大小
当前窗口大小固定为 600×700 像素。

**修改方法**：
- 编辑 `main.py` 中的 `SCREEN_WIDTH` 和 `SCREEN_HEIGHT`
- 同时调整 `ui.py` 中的卡片位置计算逻辑
